### EventBus 架构

```
SharedContext（持久化 ctx）
  └── task / search / thinking / memory / session / system

OpenClawEventBus extends EventEmitter
  ├── register(skill, handlers)  — 事件 → handler 映射
  ├── emit(event, data, targetSkill)  — skill-gating 版
  ├── canTransition(current, next)  — 状态机
  └── EVENTS 常量（15个事件）

_wrapHandler(skillName, eventType, handler)
  └── 过滤：event.target !== skillName → 跳过
  └── 附加：event._result = handler 返回值
```

**14 个事件：** user.message / skill.register / skill.match / skill.call / context.update / state.transition / memory.store / memory.query / memory.recall / memory.result / search.execute / search.result / search.cached / task.execute / task.completed / task.failed / system.ready / system.error

### 修复记录

| 问题 | 原因 | 修复 |
|------|------|------|
| 两个 Handler 同时响应同一事件 | 缺少 skill-gating | `emit(type, data, targetSkill)` + `_wrapHandler` 过滤 |
| Handler 结果返回 event 而非实际结果 | `_wrapHandler` 未传递 handler 返回值 | 在 `event._result` 附加 handler 返回值 |
| `success: !handlerResult?.success ?? true` 逻辑错 | 双重取反 | 改为 `handlerResult?.success ?? true` |
| execSync 数组形式返回 Buffer | Node.js 数组形式不用 shell | 改用 shell string form |
| TASK_CREATE 未定义 | event-bus.js 漏了 | 添加 `TASK_CREATE: 'task.create'` |
| local-search 输出去人类可读文本 | `require()` 被当成 CLI 执行 | 改用直接 HTTP 调用 `localhost:18790` |
| search-router.js 缓存统计失效 | 异步化后 stderr 收集不完整 | 捕获 stderr 到 result 对象 |

## 今日教训

- `execSync(cmd, args[])` 数组形式在 Node.js 中不使用 shell，stdout 是 Buffer
- `!x?.success ?? true` 的 operator precedence：先 `!` 再 `??`，结果永远为 true
- Skill-gating 的关键是 `emit(target)` 时指定目标，handler 用 `event.target` 过滤
