## 下午 ~14:00 VCPToolBox 安装学习

### 系统定位
VCPToolBox = 多 Agent 协作中间件（Node.js + Express + WebSocket）
定位类似 AgentSymphony，但更偏向"工具+记忆+路由"的工程化实现

### 核心架构
- **TagMemoEngine**: 四级持久化记忆系统
  - 全局 / Agent / 会话 / 消息四级 context
  - 向量检索(RAG) + 标签记忆
  - hnswlib-node（层次可导航小世界图）做向量索引
  - 支持 ollama / openai / dashscope 嵌入模型
- **Plugin**: 插件生命周期管理（加载/卸载/启用禁用）
  - 内置: SearchPlugin, CalculatorPlugin, ImageGenPlugin
  - 开发者可自定义插件
- **Agent**: 单个 Agent 实例管理
- **Router**: 语义路由（基于配置规则的 AI 模型选择）
- **ToolManager**: 工具注册与调用
- **WorkerPool**: 多模型并发请求池

### 对话协议
- 支持 SillyTavern、WebOpenUI 客户端
- WebSocket + REST 双接口
- 上下文自动注入（memory + global config）

### 记忆系统亮点
TagMemoEngine 的分级设计很值得借鉴：
- 全局记忆：跨 Agent 共享
- Agent 级：单个 Agent 的上下文
- 会话级：每次对话的临时状态
- 消息级：精确到每条消息的 metadata

这比 AgentSymphony 的 memory skill 更细粒度，适合复杂的多 Agent 协作场景。

### 依赖
- Node.js, Express, WebSocket
- better-sqlite3 (SQLite 绑定)
- hnswlib-node (向量索引)
- puppeteer (浏览器自动化)
- ollama / openai / dashscope SDK

### 配置文件
- config.env（配置数据库路径、端口、模型供应商等）
- SemanticModelRouter.json（模型路由规则）

### VCPChat（Electron 桌面客户端）
- 语音对话 + 桌面 Agent 操控
- Rust 音频引擎
- ⚠️ 仅支持 Windows/macOS，Linux 不支持

### 关键文件
- server.js - 入口，Express + WebSocket 服务器
- TagMemoEngine.js - 记忆引擎核心
- Plugin.js - 插件系统
- EmbeddingUtils.js - 向量嵌入工具
- SemanticModelRouter.json - 模型路由规则
- Router.js - 请求路由

### 安装进度（子 agent 在处理）
- 依赖: better-sqlite3, hnswlib-node, puppeteer
- 需要 node-gyp / gcc 编译环境
- npm install 在跑
