**根因**: 
- Gateway restart / webchat 断连会创建新 session
- 新 session 没有 prior context
- MEMORY.md 是 curated memory (过滤过), 不是 working memory

**解法** (3 件套):

### 1. `tools/v-snapshot.py` (14.3 KB)
- `collect`: 服务状态 (5 端口 + PID + uptime + http code) + workspace (cwd + git dirty/ahead/last sha) + active session (name/size/mtime) + active context (从 memory 文件提 last_decisions / next_actions)
- `save`: atomic write (tmp + rename) → `.v-snapshot/latest.json`, 同步写 timestamped 历史
- `status`: 人类可读打印, emoji 状态码
- `load` / `history` / `clean` (7 天 rotation)

### 2. `v-services-watchdog.sh` 集成 (5 分钟一次)
- 加 `SNAPSHOT_EVERY_N=10` (30s × 10 = 5min)
- 跟 watchdog 共用进程, 不开新 cron / timer
- 触发源: `watchdog_5min` 标记, 方便追溯

### 3. `BOOT.md` + `AGENTS.md recovery protocol`
- BOOT.md: boot-md hook 在 gateway startup 跑 `v-snapshot save + status`
- AGENTS.md: 加 "V-Snapshot Recovery (transcript 丢失场景)" 节, 教 V 自己:
  1. 看到 [系统消息] / 60s 内新 session → 立即跑 status
  2. 报告时说 "从 snapshot 恢复, 不是我记得"
  3. **不假装记得**, 让浮光确认

## 20:13 Watchdog 修复 + SOP #30 立碑 (false-positive + 双 watchdog)

**问题**: 2 个相关坑
1. **双 watchdog 并行跑** — 我之前 nohup 起的孤儿 (PID 8191) 跟 systemd 起的 (PID 8344) 同时跑, 同一秒都 save snapshot → log 里 20:08:48 出现两行 ✅ saved
2. **systemd restart 时的子进程竞态** — `systemctl restart v-services-restart` 拉起新 watchdog PID 时, 5 个服务的 child process 被一起 kill. 新 watchdog 第一圈 check → 看到 5 端口全 DOWN → restart 全部 → 眼看不眼的服务都临时中断一次

**修法**:
1. **Kill 孤儿 8191** — 以后 restart watchdog 用 `sudo -u agent sudo systemctl restart v-services-restart`, 不手动 nohup
2. **v-snapshot history 文件名加 ms 后缀** — 同秒多进程不覆盖 (`{ts}-{ms%1000:03d}.json`)
3. **watchdog 加 GRACE_PERIOD_S=60** — 启动后 60s 只 check 不 restart, 让 systemd 把 5 服务 settle

**端到端验证 (V 20:15)**:
- 20:15:32 systemd 拉起新 PID 10887, grace 60s 开始
- 20:15:32+5s  pkill ollama (模拟 DOWN)
- 20:16:03 (grace 31s) → watchdog log: `ollama :11434 DOWN, grace-skip (let systemd settle)` ✅
- 20:16:33 (grace 61s) → watchdog log: `ollama :11434 DOWN, restarting...` → 2s 后 `restarted ✅`

### SOP #30 — 守护进程 + 并行跑 立碑 (5 永久教训)
1. **Restart daemon 用 systemd, 不用 nohup** — 手动起的进程跟 systemd 起的会并行, 双倍 save / 双倍监控
2. **Grace period 防启动竞态** — watchdog 起来后 sleep grace_seconds, 只 check 不 restart, 让 systemd 把 children settle
3. **同秒 snapshot 加 ms 后缀** — 多 watchdog 并行 / 外部 cron 都可能同秒触发, history 文件名不覆盖
4. **看 journal 跟 /tmp/log 两份** — systemd 跑用 journalctl, nohup 跑用 /tmp. 不要混
5. **kill 进程用 pkill -f, 不用手记 PID** — systemd 重启会换 PID, 手记的 PID 必猜错
