### SOP #33 — Detached HEAD / 0-commit 检测
**触发**: AgentMemory 6/15 报告 "HEAD detached" (失实, 实际是 0 commit + symlink 错位)
**检测** (5 项):
1. `git rev-parse HEAD` → `fatal: ambiguous argument 'HEAD'` = 0 commit
2. `git symbolic-ref HEAD` → `refs/heads/master` (分支存在, 0 commit)
3. `git reflog` → 空 = 0 commit 后的操作历史
4. `git status` → 报 "未跟踪的文件" (全部) = 0 commit
5. `ls -la .git/HEAD .git/refs/heads/*` = 缺文件 = 真 0 commit 仓
**根因分类**:
- (a) 全新仓, 从未 commit
- (b) symlink 错位 (AgentMemory 6/18) — git 找父级 .git, 但 cwd 不在真仓
- (c) reflog expire + 后续 reset (罕见)
**修法**: (a) 初始 commit; (b) 检查 symlink + `readlink -f`; (c) `git fsck --no-reflogs` 看 dangling objects
**教训**: 6/15 报告 "fsck + reflog 重建" 是 (a) 路径, 但实际是 (b) — 必须 L1 验证根因
